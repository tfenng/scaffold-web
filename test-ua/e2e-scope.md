# E2E Scope

## 测试目标

本轮端到端测试聚焦用户列表管理页面的核心业务链路，验证前端在真实后端环境下是否能够稳定完成列表读取、用户维护和异常处理。

## 纳入首批范围

- 用户列表加载
- 空列表与非空列表展示
- 用户创建
- 用户编辑
- 用户删除
- 删除确认与取消删除
- 表单前端校验与后端错误反馈
- 分页行为
- 列表失败后的重试
- 新版 `PATCH /api/v1/users/{id}` 局部更新语义

## PATCH 语义专项验收

- 修改单个字段时，仅该字段发生变化
- 未修改字段在保存后保持原值
- 清空 `email` 或 `birth` 后，页面重新读取应显示为空值
- 编辑表单未修改时，保存动作不可执行

## 不纳入首批范围

- 登录、鉴权、权限模型
- 跨模块跳转
- 浏览器兼容矩阵
- 性能压测
- 非 `/users` 页面
- 后端权限、审计、数据库层验证

## 环境边界

- 依赖真实后端，不采用 API mock
- 假设测试环境允许创建、修改、删除测试用户
- 不假设当前仓库已经完成 Playwright 工程接入

## 验收标准

- 首批用例覆盖 CRUD 主链路
- 至少有一组用例覆盖新版 `PATCH` 局部更新与 nullable 字段清空
- 每条数据变更用例都具备独立准备与清理策略
